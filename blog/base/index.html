WHY EXCEL GETS MESSY

為什麼 Excel 會開始混亂？

01

資料越來越分散

客戶資料、報價、跟進、庫存、項目紀錄分散在不同表格，最後沒有人知道哪一份才是最新版本。

02

欄位沒有標準

同一個狀態可能有人寫「已跟進」、有人寫「followed」、有人寫「已覆」，後續很難篩選和統計。

03

流程只存在腦入面

誰要跟進、何時再聯絡、下一步做什麼，全靠人記住。人一忙，就容易漏。

STEP 01

分清楚資料類型

資料類型用途常見欄位
客戶資料 記錄客戶身份、聯絡方式及關係狀態 客戶名稱、聯絡人、電話、Email、來源、狀態、備註
銷售 / 跟進 記錄查詢、報價、跟進及成交進度 查詢日期、需求、報價金額、跟進人、下一步、下次跟進日期
項目 / 任務 記錄工作進度、負責人及交付日期 項目名稱、階段、負責人、截止日期、進度、風險
產品 / 庫存 記錄產品資料、庫存數量及補貨狀態 產品名稱、SKU、成本、售價、庫存、供應商、補貨提醒
支援 / 查詢 記錄客戶問題、處理狀態及常見查詢 問題類型、客戶、內容、狀態、負責人、解決方法
STEP 02

每張表只回答一個問題

A
客戶表

這個客戶是誰？

  • 客戶名稱
  • 聯絡人及聯絡方法
  • 客戶來源
  • 目前狀態
B
跟進表

這次查詢到哪一步？

  • 查詢內容
  • 報價狀態
  • 下一步行動
  • 下次跟進日期
C
項目表

成交後要交付什麼？

  • 項目階段
  • 負責人
  • 截止日期
  • 風險或卡點
STEP 03

為每個流程設定清楚狀態

流程建議狀態
客戶跟進 新查詢 → 已回覆 → 已報價 → 待跟進 → 已成交 → 未成交 / 暫停
項目進度 未開始 → 進行中 → 等待客戶 → 內部檢查 → 已完成 → 暫停
庫存處理 正常 → 低庫存 → 待補貨 → 已下單 → 已入庫
客戶支援 新問題 → 處理中 → 等待回覆 → 已解決 → 需升級處理
STEP 04

把表格之間連起來

當資料開始變多，不同表格之間應該有關聯，而不是每張表各自獨立。

例子：一個客戶可以有多次查詢；一個查詢可以變成一個項目；一個項目可以產生多個任務。
01

客戶表

記錄「這個客戶是誰」

02

跟進表

記錄「這次查詢跟進到哪裏」

03

項目表

記錄「成交後要交付什麼」

STEP 05

判斷是否需要 Excel、Airtable、Web App、自動化或 AI

情況可以先用之後可升級
資料量不大，只是想集中管理 Google Sheet / Excel 表單、自動提醒、簡單 Dashboard
資料之間有關聯，例如客戶、訂單、項目 Airtable / Notion database 自動化流程、權限管理、報表
經常要重複整理、分類或總結資料 標準欄位 + 固定格式 AI 分類、AI 總結、AI 分析
員工需要登入、填表、查資料 表單 + Sheet 簡單 Web App / 內部工具
